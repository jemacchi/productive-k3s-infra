# Open vs. Pro Boundary

The public repository should maximize adoption by making the project easy to evaluate and reuse.

Public scope:

- Multipass local environments
- basic AWS single-node setup
- basic on-premises SSH setup
- generic OpenTofu modules
- generic Ansible roles

Private/commercial scope:

- HA cloud clusters
- hardened networking
- managed customer inventories
- automated upgrades
- backup and restore workflows
- production validation packs
- packaged application stacks
