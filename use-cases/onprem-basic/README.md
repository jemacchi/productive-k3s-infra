# On-Premises Basic Use Case

Basic on-premises provisioning for existing machines reachable over SSH.

This use case assumes the user already has machines available and wants to install Productive K3S without cloud provisioning.

Public scope:

- static Ansible inventory
- SSH-based preparation
- single-node or simple server/agent layouts
- basic validation

Advanced HA layouts, upgrade automation, backup/restore procedures, hardened access patterns, and customer-specific inventories belong in `productive-k3s-infra-pro`.
